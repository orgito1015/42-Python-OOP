# 🌱 42 Python OOP — Garden-Themed Learning Series

Welcome to the **42 Python OOP** repository! This is a structured, garden-themed learning curriculum that guides you from your very first Python function all the way through advanced Object-Oriented Programming patterns. Every concept is taught through the lens of a virtual garden, making abstract ideas feel natural and memorable.

---

## 📚 Projects Overview

| Project | Track | Exercises | Description |
|---------|-------|-----------|-------------|
| [Growing Code](#-growing-code) | Functional Programming Foundations | 8 | Functions, user input, conditionals, loops, and recursion |
| [CodeCultivation](#-codecultivation) | Object-Oriented Programming | 7 | Classes, encapsulation, inheritance, polymorphism, and advanced patterns |
| [Garden Guardian](#-garden-guardian) | Full Application | — | A complete garden management application (external repo) |

## 🌿 Growing Code

**Track:** Functional Programming Foundations  
**Location:** [`Growing Code/`](./Growing%20Code/)

A beginner-friendly project that introduces the core building blocks of Python programming using simple garden-management scenarios. Each exercise isolates a single concept to build a solid foundation before moving into OOP.

| Exercise | File | Concept |
|----------|------|---------|
| ex0 | `ft_hello_garden.py` | Defining and calling functions |
| ex1 | `ft_plot_area.py` | User input, type conversion, arithmetic |
| ex2 | `ft_harvest_total.py` | Accumulating values across multiple inputs |
| ex3 | `ft_plant_age.py` | Conditional logic (`if/else`) |
| ex4 | `ft_water_reminder.py` | Practical conditionals for decision-making |
| ex5 | `ft_count_harvest_iterative.py` / `ft_count_harvest_recursive.py` | Loops vs. recursion — two ways to solve the same problem |
| ex6 | `ft_garden_summary.py` | Combining input, variables, and formatted output |
| ex7 | `ft_seed_inventory.py` | Function parameters, type hints, and multi-branch conditionals |

[→ Full details in Growing Code README](./Growing%20Code/README.md)

## 🌳 CodeCultivation

**Track:** Object-Oriented Programming  
**Location:** [`CodeCultivation/`](./CodeCultivation/)

A progressive OOP curriculum that starts by introducing classes and gradually layers on encapsulation, inheritance, polymorphism, and advanced patterns like nested classes, class methods, and static methods.

| Exercise | File | Concept |
|----------|------|---------|
| ex0 | `ft_garden_intro.py` | Python program structure, variables, f-strings |
| ex1 | `ft_garden_data.py` | Class definition, `__init__`, instance attributes |
| ex2 | `ft_plant_growth.py` | Instance methods, state management |
| ex3 | `ft_plant_factory.py` | Object creation patterns, lists of objects |
| ex4 | `ft_garden_security.py` | Encapsulation, private attributes, getters/setters, validation |
| ex5 | `ft_plant_types.py` | Inheritance, `super()`, method overriding, polymorphism |
| ex6 | `ft_garden_analytics.py` | Nested classes, inheritance chains, `@classmethod`, `@staticmethod` |

[→ Full details in CodeCultivation README](./CodeCultivation/README.md)

## 🔧 Requirements

- Python 3.x
- No external dependencies — all exercises use the Python standard library only

---

## 🚀 Running the Exercises

Each exercise is a self-contained Python file. Run any file directly from your terminal:

```bash
# Example: run a CodeCultivation exercise
python3 CodeCultivation/ex2/ft_plant_growth.py

# Example: run a Growing Code exercise
python3 "Growing Code/ex1/ft_plot_area.py"
```

## 🌱 Theme

The garden metaphor is used consistently throughout all three projects. Plants grow, get watered, get harvested, and compete for prizes — all while teaching real programming concepts. This cohesive theme makes it easier to focus on the code rather than the domain.
